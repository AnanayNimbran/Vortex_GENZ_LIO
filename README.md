# Vortex_GENZ_LIO (INDEPENDENT C++ / ROS 2 REPRODUCTION)

CLEAN-ROOM REIMPLEMENTATION OF **"GENZ-LIO: GENERALIZABLE LIDAR-INERTIAL ODOMETRY
BEYOND CONFINED–OPEN BOUNDARIES"** (LEE ET AL., ARXIV:2603.16273V2). THIS IS *NOT*
THE AUTHORS' OFFICIAL CODE; IT IS RECONSTRUCTED FROM THE EQUATIONS AND ALGORITHM
BOXES IN THE PAPER. ALL INTELLECTUAL CREDIT FOR THE UNDERLYING ARCHITECTURE (SCALE-AWARE ADAPTIVE VOXELIZATION, HYBRID-METRIC STATE UPDATE) BELONGS ENTIRELY TO THE ORIGINAL AUTHORS. IF YOU USE THIS IMPLEMENTATION IN ACADEMIC WORK, PLEASE CITE THEIR PAPER.


## WHAT IS IMPLEMENTED (ALL THREE CONTRIBUTIONS, NO STUBS IN THE CORE MATH)

| PAPER SECTION | MODULE | WHERE |
|---|---|---|
| IV — SCALE-AWARE ADAPTIVE VOXELIZATION (PD CONTROL + SENSITIVITY-INFORMED GAIN SCHEDULING, EQ.1–8, ALG.1) | `ADAPTIVEVOXELIZER` | `GENZ_LIO.HPP` / `.CPP` |
| V-B — VOXEL-PRUNED CORRESPONDENCE SEARCH (CANDIDATE-VOXEL SELECTION FIG.5, DISTANCE-TO-VOXEL FIG.6, TWO-STAGE PRUNING ALG.2) | `VOXELMAP` | `GENZ_LIO.HPP` / `.CPP` |
| V — HYBRID-METRIC ESIKF STATE UPDATE (POINT-TO-PLANE EQ.13–19 + POINT-TO-POINT L2 EQ.20–30, FUSED MAP UPDATE EQ.31–35, ALG.3) | `ESIKF` | `GENZ_LIO.HPP` / `.CPP` |

SUPPORTING MACHINERY THE PAPER EXPLICITLY DELEGATES TO PRIOR WORK IS IMPLEMENTED
FROM THE CITED REFERENCE AND LABELLED IN COMMENTS:
* PER-POINT / PLANE UNCERTAINTY → YUAN ET AL. *VOXELMAP* [11];
* ESIKF FORWARD PROPAGATION + REPARAMETRIZATION `J` → XU ET AL. *FAST-LIO2* [6].

## BUILD (ROS 2 HUMBLE / IRON / JAZZY, UBUNTU 22.04+)

BUILD YOUR WORKSPACE.

## RUN

```BASH
ROS2 LAUNCH GENZ_LIO GENZ_LIO.LAUNCH.PY          # ALSO OPENS RVIZ2 (RVIZ:=FALSE TO SKIP)
# REPLAYING A BAG? USE SIM TIME:
ROS2 LAUNCH GENZ_LIO GENZ_LIO.LAUNCH.PY USE_SIM_TIME:=TRUE
# CUSTOM PARAMETERS:
ROS2 LAUNCH GENZ_LIO GENZ_LIO.LAUNCH.PY \
  CONFIG:=$(ROS2 PKG PREFIX GENZ_LIO)/SHARE/GENZ_LIO/CONFIG/GENZ_LIO.YAML
```
PUBLISHES `~/ODOMETRY`, `~/PATH`, `~/CLOUD_REGISTERED` (WORLD) PLUS
`~/CLOUD_REGISTERED_BODY` (WHEN `SCAN_BODYFRAME_PUB_EN`), AND THE `ODOM→BASE_LINK`
TF (WHEN `PUBLISH_TF`). ON SHUTDOWN, THE VOXEL MAP IS WRITTEN TO `PCD_SAVE_PATH`
IF `PCD_SAVE_EN` IS SET.

## TESTS

`COLCON TEST` RUNS `TEST/TEST_GENZ_MATH.CPP` (GOOGLETEST): SO(3) ROUND-TRIPS,
BOXPLUS/BOXMINUS CONSISTENCY, THE PD CONTROLLER (SETPOINT MONOTONICITY, CLAMPING,
ERROR RESPONSE), AND THE VOXEL-PRUNING GEOMETRY (FIG.5 NEIGHBOUR COUNTS, FIG.6
DISTANCES). THE ESTIMATOR WAS ADDITIONALLY VALIDATED TO (A) REMAIN AT GROUND
TRUTH WHEN INITIALIZED THERE AND (B) CONVERGE BACK TO GROUND TRUTH FROM A
REALISTIC IMU-PRIOR OFFSET USING POINT-TO-PLANE **AND** POINT-TO-POINT FUSION.

## KNOWN CONSIDERATIONS

* ITERATED-EKF / ICP CORRESPONDENCE SEARCH CAN OSCILLATE UNDER *VERY LARGE* INITIAL
  OFFSETS IN PERFECTLY SYMMETRIC SCENES; IN NORMAL OPERATION THE PER-SCAN IMU PRIOR
  KEEPS THE LINEARIZATION POINT CLOSE, WHICH IS THE VALIDATED REGIME.
